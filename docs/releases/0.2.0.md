# Kllama 0.2.0

Kllama began as a small educational project to help students understand how local generative AI systems work in practice. Version 0.2.0 is the maintenance refresh that brings the project up to date while preserving that original spirit.

This release makes the repository easier to trust, easier to run, and easier to extend. The app remains intentionally lightweight, but the surrounding engineering now better reflects a maintained project.

## Highlights

- Refreshed the Streamlit app for current Ollama and Streamlit workflows.
- Added model refresh, generation controls, system prompt steering, and transcript download.
- Extracted reusable helper logic into a dedicated module.
- Replaced the old live-only test script with offline pytest coverage.
- Added GitHub Actions CI for modern Python versions.
- Updated documentation to explain the educational purpose and ongoing maintenance of the project.

## Why This Release Matters

Kllama was already aligned with ideas that later became central in GenAI tooling: local inference, privacy-aware experimentation, small composable apps, and transparent prompt-driven workflows. This release makes that case more clearly for future readers, students, and collaborators.

## Included In This Release

### App Improvements

- Better Ollama connection handling.
- Cleaner model selection flow.
- Sidebar controls for generation parameters.
- Downloadable Markdown chat transcripts.

### Project Hygiene

- Packaged with `pyproject.toml`.
- CI added for push and pull request validation.
- VS Code workspace configured for the local virtual environment.
- Dependency constraints updated for current versions.

### Testing

- Pure helper functions covered by pytest.
- Streamlit app smoke test scaffold added with Ollama mocked.

## Breaking Changes

- None intended. The project is still launched with `streamlit run kllama.py`.

## Upgrade Notes

If you are pulling the latest version of the repository, recreate or refresh your virtual environment and reinstall dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
```

## Looking Ahead

The next maintenance steps are straightforward: add visual demo assets, expand app-level testing, and continue documenting the project as a long-running teaching artifact for practical GenAI.
