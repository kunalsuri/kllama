# Code of Conduct

Kllama is a small educational open source project. Contributors, users, and maintainers are expected to keep the project respectful, constructive, and safe for others.

## Expected Behavior

- Be respectful in language, tone, and technical disagreement.
- Assume good intent, but give clear and direct feedback.
- Focus review comments on the code, documentation, behavior, and user impact.
- Welcome contributors with different backgrounds, skill levels, and experiences.
- Share context when making decisions so others can learn from the discussion.

## Unacceptable Behavior

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory or hateful language.
- Trolling, deliberate disruption, or bad-faith engagement.
- Publishing someone else's private information without permission.
- Repeatedly ignoring project boundaries, review feedback, or repository rules.

## Scope

This Code of Conduct applies to repository discussions, issues, pull requests, code review, and other project-related public interactions.

## Maintainer Responsibilities

Maintainers may edit, remove, reject, or moderate content that violates this Code of Conduct. They may also temporarily or permanently restrict participation when necessary to protect the project and its contributors.

## Reporting

If you experience or witness unacceptable behavior, report it to the repository maintainer through GitHub contact channels. Include links, screenshots, or other concrete context when possible.

All reports should be handled as discreetly and fairly as practical.
